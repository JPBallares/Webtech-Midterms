W W W (World Wide Web)

Overview :

WWW stands for World Wide Web is a collection of web resources and applications that identified by Uniform Resource Locators (URLs), interlinked by hypertext links that can be access via the internet. World Wide Web was invented by Tim Berners-Lee and co-inventor Robert Cailliau in 1989 at CERN in Geneva. The main goal of Tim Berners-Lee for inventing WWW is for the researcher around the world for them share information.

W W W (World Wide Web)

Internet :

- Global network of networks.
- A collection of hardware and software (e.g. Hosts, Media, Protocols).
- Uses internet protocol suite (TCP/IP) to link devices worldwide.

H T T P (Hypertext Transfer Protocol)

Overview :

HTTP stands for Hypertext Transfer Protocol that is used by the web. It defines how the information is transmitted and formatted. HTTP is an application layer communications protocol used to access resources (Hypertext/Hypermedia) on the World Wide Web. Develop by Tim Berners-Lee in 1989 at CERN. Port 80 is the default port.

H T T P (Hypertext Transfer Protocol)

HTTP Message :

HTTP Message is use to communicate between the client and the server. The client sends a request message and the server will respond sending a response message corresponding to the request of the client. The Request messages is automatically created when accessing resources using browsers, but it can be create manually. The Response message is also automatically created but it will depend on the server's configuration.

H T T P (Hypertext Transfer Protocol)

Request Message :

Request Message is a message sent by the client to the server in order to access resources. It is compose of Request Line, Message Headers, Empty Line (CRLF) , and Pay Load or Body.

Request Message Parts :


Request Line

- Compose of request method, request target, and protocol version.

Request Method

- It informs the server what the client wants to do with the resources.

Request Target

- The URI or Uniform Resource Identifier that identifies the resources that the client wants to access is located ( asterisk "*" can be used to refer to the server itself ).

Protocol Version

- Provides the version of the HTTP that the client uses, so that the server knows what HTTP version to return to the client. (E.g HTTP/1.1).

Request Message Headers

- This allows client to provide more information to the server so that the server will be able to know what the client is capable of doing or what the client can handle. This information is placed in Request Header Fields.

Empty Line

- Indicates that the end of the Header Fields has been reached.

Message Body or Payload

- An optional part of a request message that contains a resource which can be sent to the server (using PUT method) or updated by the server.

H T T P (Hypertext Transfer Protocol)

Response Message :

Response Message is a message created by the server to answer the request of the client. It is compose Status Line, Message Headers, Empty Line (CRLF), and the Message Body or Payload.

Response Message Parts :


Status Line

- Compose of Protocol Version, Status Code, and Status Description.

Protocol Version

- Provides the HTTP version of the Response Message.

Status Code

- Code compose of 3 digit that describes what happen to the request of the client, if it is successfulor failure. Some examples are 200, 404, 304.

Status Description

- A brief description of the Status Code (human-readable). Examples: Ok, Not Found, Not modified. Example of Status Line: HTTP/1.1 200 OK, HTTP/1.1 404 Not Found, HTTP/1.1 304 Not Modified

H T T P (Hypertext Transfer Protocol)

GET :

- Retrieve full resources from the server, can also be used to submit information but it is not advisable because the information can be seen on the URI.

H T T P (Hypertext Transfer Protocol)

HEAD :

- Retrieve meta-data information of the resources, without the message body or the content of the resources.

H T T P (Hypertext Transfer Protocol)

PUT :

- Submit new resources on the server, or edit an existing resource in the server, the resources to be sent or modified is placed in the message body.

H T T P (Hypertext Transfer Protocol)

DELETE :

- Remove the resources that from the server that is identified by the URI specified by the client. This will fail if the server doesn't have the resource identified by the client.

H T T P (Hypertext Transfer Protocol)

OPTIONS :

- Shows the possible method that can be applied to the resources identified by the URI. This can also be applied to the server itself ( by using * as the target ) this will show the allowed method by the server.

H T T P (Hypertext Transfer Protocol)

TRACE :

- This will allow to track down where did the request message pass before getting into the server, this can also help client to see the exact message that arrives at the server.

H T T P (Hypertext Transfer Protocol)

CONNECT :

- Use proxy to establish tunnel to the remote server. It usually is being used in secured connection.

H T T P (Hypertext Transfer Protocol)

Safe Methods :

- This methods are only used as retrieval of resources ( read-only ), therefor this request message will not affect the server or the other client accessing the same resources and can be considered safe. Methods that are unsafe should be disabled, except for some special cases, by the server to avoid client from doing unnecessary action and the server can work properly.

H T T P (Hypertext Transfer Protocol)

Idempotent Method :

- This methods are considered idempotent because the expected result of this request methods to the server will be the same even if it is requested single time or it is requested multiple times(except caching and errors). This can be identified easily because when failure occurs in the communication the client can just resend the request to the server and expect that the result will be done.

H T T P (Hypertext Transfer Protocol)

Cacheable Methods :

- It is possible to store the current request and reuse it when other client request with the same request message. In some cases POST request method will not be cache depending on the configuration of the server.

H T T P (Hypertext Transfer Protocol)

General Headers :

- Is being used by the both client and server in sending messages ( request and response messages ) and has no relation to the body is is being sent. Some examples are Date, Cache Control, Pragma, Connection, Transfer Encoding, Upgrade, and Warning.

H T T P (Hypertext Transfer Protocol)

Request Headers :

- Provide information about the client or the resource that the client want to access. Some examples are Accept, Accept-*, If-*, Cookie, User-Agent, Host and Referer.

H T T P (Hypertext Transfer Protocol)

Response Headers :

- Provide information about the response being sent or the information about the server. Some examples are Age, Location, and Server.

H T T P (Hypertext Transfer Protocol)

Entity Headers :

- Is used to provide information about the message body or payload, like content length and content type. Some examples are Content-Type, Content-Length, Content-Language and Content-Encoding.

H T T P (Hypertext Transfer Protocol)

Content Negotiation :

- allows HTTP clients to negotiate for the resources that it prefer using and the server can provide for the available

Accept

- Provide the type of the resources to be accessed by the client.

Accept-Charset

- Provide the information on character sets that can be interpreted by the client.

Accept-Encoding

- Provide the type of encoding that can be interpreted by the client, this usually refers to the compression that the server can handle.

Accept-Language

- Provide the expected language to be accessed by the client. The client should provide a sequence of language sin case the first language is not available.

H T T P (Hypertext Transfer Protocol)

Connection Management :


Connection

- Manage the network connection to open or close the current communication after it is finished.

Keep-Alive

- Set how long a connection can stay open after the communication is finished.

H T T P (Hypertext Transfer Protocol)

Range Requests :

- Allow the client to access part of the resources.

Range

- Included in the client request, to specify a partial request of the resource.

Accept-Range

- Indicates if the server allows partial request on the resources.

If-Range

- Provides condition for the range request, range request will only be successful if the condition is met.

Content-Range

- Specify part of the resources to be accessed.

H T T P (Hypertext Transfer Protocol)

Authentication and Authorization :

- Allows the server to restrict resources that can be accessed by the client, and allows the client to provide their credentials to access the file.

WWW-Authenticate

- Describes how the client can access the resources.

Authorization

- Provides the type of authentication (Basic Authentication, Digest Authentication) and the client credentials. The credentials is usually encrypted.

Proxy-Authenticate

- Describes how the client can be access the resources in a Proxy Server.

Proxy-Authorization

- Provides the type of authentication ( Basic Authentication, Digest Authentication ) and the client credentials in a proxy server.

H T T P (Hypertext Transfer Protocol)

Conditionals :

- Specify conditions that must be met in order for the client to access the resources or in order for the server to send the response.

Last-Modified

- Provides the date of modification of the resources, it can be used to validate the client's cache and the server's resource.

Etag

- This uniquely identify the resource's version, this will be used as a condition when checking for client's cache and the server's resource.

If-Match

- A condition that checks if the Etags of the client's caches matches the Etags of the resources being accessed before doing the operation.

If-None-Match

- A condition that checks if the Etags of the client's caches didn't match any of the Etags in the servers resources. Usually used to update the cache of the client or to prevent duplication of resources.

If-Modified-Since

- A condition that checks if the resource is modified since a given date. This can help to determine if the cache is out of date.

If-Unmodified-Since

-A condition that checks if the resources is not modified since a given date.

- The request was successful.

- Successfully processed the request but no contents returned.

- Successfully processed the request but no contents returned.

- DAV agreement have formerly been counted and listed in an earlier reply to the request and aren't being incorporated any more.

- Clients must take additional action to complete the request.

300 Multiple Choices

- Indicates multiple options for the resource that the client may follow.

301 Moved Permanently

- This and all future requests should be directed to the selected URL.

302 Found

- The requested resource resides temporarily under a different URI.

303 See Other ( since HTTP/1.1 )

- The response to the request can be found under a different URI and SHOULD be retrieved using a GET method on that resource. The new URI is not a substitute reference for the originally requested resource.

C S S (Cascading Style Sheet)

Overview :

Cascading Style Sheets (CSS) is a language used to specify presentation, style or layout of a markup languages (HTML, XHTML, XML, SVG) but it is more common on web pages. The presentation, style or layout may refer to font, color, size, and position of the contents. It can also be used in adding animation to a web pages. It was developed by Håkon Wium Lie and Bert Bos. The idea for using CSS is to separate the presentational aspect from markup languages.

C S S (Cascading Style Sheet)

CSS Level 1 (CSS 1) :

The first CSS that is published on December 1996, and became an official W3C Recommendation. The features of CSS Level 1 are font properties, changing colors of elements like font color and background color, text attributes, alignment of contents (text, images, tables and others), the concept of border, margin, padding and positioning and unique identification and generic classification of groups of attributes.

C S S (Cascading Style Sheet)

CSS Level 2 (CSS 2) :

Published on May 1998 by W3C as a Recommendation. Added functions mostly for positioning layout like position relative, absolute, and fixed, also added z-index which is use to position element above or under other elements. It also added media types which can be use to display web pages in different platform, aural style sheets which allows audio or sound presentation, bidirectional text, and new font properties.

There is also a CSS Level 2 Revision 1 (CSS 2.1), which is released to fix bugs and improve compatibility of in CSS 2 which is released on June 2011

C S S (Cascading Style Sheet)

CSS Level 3 (CSS 3) :

In CSS 3, there are many features that is introduced, included are new selectors (:target, nth-child(n), etc.), new properties (flexbox, grid, border-radius, etc), multiple background image, rounded border using border-radius, new text module, and new color module like the rgba which includes the transparency of the color.

C S S (Cascading Style Sheet)

Origin of HTML Stylesheets :


Author Style

It is a style sheets specified by the developer of the website. Which can be categorized into:

  1. Inline - placed in a specific element by using style attribute.

    <p style="color: white; background-color: rgba(0, 0, 0, 0.7);"> </p>

  2. Embedded or Internal - placed in the head of html using the style element.

    <style> </style>

  3. External - placed in the head of html using the link element.

    <link rel="stylesheet" href="">

Note: The categories of author style above is aranged (Highest to Lowest priority) when the author specified more than one style for specific element, the one that will apply is the inline since it has higher priority.

User Style

A style sheets specified by the user for the website, it is usually done by having some plugins in the browser, although some browsers support it. It is helpful for person with eye problems.

User-Agent Style

Also known as default style sheets, it is provided by the browser that is being used to access the website. Different browsers have different style sheets to present the website. It is used when there is no other style sheets are specified for specific element of HTML.

!important Declaration

A property can be declared !important meaning it is higher priority than the other property, it will not be overridden by property declared normal even if it is declared after the !important declaration. In some cases there might be two !important declaration for the same property, then we can look into the order of the origin to see which will be applied.

Precedence of Origin (Cascading)

When there are collision between properties, meaning the property is being declared multiply times in different style sheets, there is a order in which style sheet will be applied for the property (Highest to Lowest priority):

  1. Transition declarations
  2. User agent important declarations
  3. User important declarations
  4. Override important declarations
  5. Author important declarations
  6. Animation declarations
  7. Override normal declarations
  8. Author normal declarations
  9. User normal declarations
  10. User agent normal declarations

Note: When a property is declared in the same origin then the latest declaration will be one one that will apply.

C S S (Cascading Style Sheet)

CSS Statements :

CSS statement are statements that is being rendered by the browser to provide styles or lay-out for elements of HTML or the HTML itself. This statements can be categorized into two:

  • Ruleset is a combination of selectors and declaration block, considered as main building block of CSS.
  • At-Rules begins with @ followed by an identifier, it tells the CSS what to do and how it should be done. Each of the identifier has its own way of creating a statement.

CSS Ruleset

Compose of selector which represents the element of the HTML to apply the style, declaration block which is compose of declarations.

CSS Declaration Block

Compose of CSS declarations enclosed by curly braces ({ }) which is applied for specific elements of HTML. For inline CSS using style attribute, curly braces are removed because it can only be applied for the element with the style attribute.

CSS Declaration

CSS declaration is the combination of properties and its value in order to apply the style to elements of HTML. It is composed of property, colon(:) that serves as separator, value, and semi-colon(;) which tells that the declaration ended or in order to separate from other declarations.

  • The property refers to the feature that will be applied for the design or style, like color, background, border, and many more.
  • The value is used to specify how the property will act when rendered by the browser, like for property color it can have values white, red, or black.

C S S (Cascading Style Sheet)

CSS At-rules :


Regular At-rules

Regular At-rules follows regular syntax @KEYWORD followed by the rule to specifiy.

  • @charset - used to specify the character set to be used when rendering the website.
  • @import - used to include another external CSS in a CSS, must be inserted at topmost part of the stylesheet.
  • @namespace - allows the use of other namespaces from XML or SVG.

Nested At-rules

Nested At-rules contains CSS statements inside its block, this statements will only apply if certain condition has been met by the user or the browser. The CSS statments are contained in curly braces ({ }) which follows after the @KEYWORD.

  • @media - allows to apply styles to a specific media query (eg. screen sizes)
  • @supports - can be used to check if browser supports a specific feature before applying the styles.
  • @document - allows to apply styles to a specific page, this styles will be ignored by the other pages of the website.
  • @page - use to layout the printing of the page.
  • @font-face - use to add new font that can be used in styling the website.
  • @keyframes - use to specify keyframes for an animation which can be applied to a selector.
  • @viewport - currently at the Working Draft stage
  • @counter-style - at the Candidate Recommendation stage, but only implemented in Gecko as of writing
  • @font-feature-values - at the Candidate Recommendation stage, but only implemented in Gecko as of writing

C S S (Cascading Style Sheet)

CSS Selectors :

A selector is a expression which is used to select which HTML elements will the CSS statements will apply.

Simple Selectors


  • Universal - used to apply styles in all element of HTML

    Example: *, .classname

  • Type Selector - name of the element, styles will apply to all of the element with that name.

    Example: h1, p, body, head

  • ID Selector - elements with an attribute id, styles will apply if the id attribute matches the id selector.

    Example: #main, #navigation, #chapter1, h1#mainheading

  • Class Selector - elements with an attribute class, styles will apply if the class attribute matches the class selector.

    Example: h1.normal, .classname, p.content, p.lead.quote

  • Attribute Selector - elements with a specific attribute, or attribute value will be selected to apply style.

    Example: img[alt], nav[id=main], [id=main], a[href$=.png], a[href^=https], p[lang|=en], a[href*=google]

Combinators


  • Adjacent Sibling Combinator - uses + symbol,selects element that follows right after the specified element and they both have the same parent.

    Example: h3 + ul, h2 + p

  • General Sibling Combinator - uses ~ symbol,selects element that follows after the specified element but not immediately, and they both have the same parent. Example: h2 ~ div, h3 ~ ol
  • Child Combinator - uses > symbol, selects element that are children of the specified element (not including those that are under another element). Example: ul > li , all li elements that are directly inside the ul is selected, but if the li is contained in a div even if the div is contained in ul, it will not be included.
  • Descendant Combinator - uses a white space, selects element that are inside the specified element even if it is contained in other element if the element is inside the specified element.

    Example: article div selects all div that are inside the article.

CSS Rule Precedence :

an HTML element may be the subject of the multiple style rules

  • when such rules target different properties their effects cascade (i.e, are combined)
  • when the styles involve the same property, they conflict, and must be resolved such that one style is applied
  • resolution
    1. by origin and importance
    2. by specificity - more specific selector will be applied.
      The selectors are arranged in order of highest to lowest specificity:
      1. ID selectors
      2. Class selectors
      3. Type Selectors

      Note: If the selector is applied in an inline style, the effect of the properties of that selector will be prioritized.

    3. by order - latest declaration will apply

CSS Colors :

This is a CSS module that deals with colors, color types, color blending, opacity, and how you can a[[ly these colors and effects to HTML content. Not all CSS properties that take a <color> as a value are part of this module, but they do depend upon it

Event Registration


EventTarget

- This interface lets you register the removal and addition of EventListeners

Types


  1. inline event attribute - event is registered as an attribute of element.

    <button type="button" onclick="function(){};">Click Me</button>

  2. PreDOM L2 - allows only one handler.

    <script>
      document.getElementById("button").onclick = function;
    </script>
    <button type="button" id="button">Click Me</button>

  3. DOM L2 - allows multiple handler.

    <script>
      document.getElementById("button").addEventListener("click", function);
      // to use capture phase
      document.getElementById("button").addEventListener("click", function, true);
    </script>
    <button type="button" id="button">Click Me</button>

    You can also remove and dispatch own event using removeEventListener() and dispatchEvent().

  4. Microsoft-specific, IE (v6-v10) - also allows multiple handler.

    <script>
      document.getElementById("button").attachEvent("onclick", function);
    </script>
    <button type="button" id="button">Click Me</button>

    Here the only phase available is bubble phase so the third parameter is removed. You can also remove and dispatch own event using detachEvent() and fireEvent().

Event Propagation Order


  1. Capture Phase

    Event capture is a process that is registered in the eventListener that can be intercepted by the event target's ancestor. It operates from the root event down to the event target.

  2. Target Phase

    Event target is the reference to an object that was originally dispatched the event

  3. Bubbling Phase

    Event Bubbling is opposite of event capture. the event target can trigger its ancestors that it can find. It operates form the event target up to the root tree.

Event Objects Properties and Methods

Pakilagyan nalang ng definition

  • eventPhase - has a number value which indicates the event phase of the event. Numbers correspond to a event phase.

    0 - None
    1 - Capture Phase
    2 - At Target
    3 - Bubble Phase

  • bubbles - Has a boolean value which indicates if the event is bubble phase or not (capture phase)
  • target (srcElement) - The main element which actually triggered the event.
  • currentTarget - This is where the event is currently being handled.
  • preventDefault() - Prevent a default event from executing, for example an anchor element will redirect you to a link, but using preventDefault() it can be stopped from going to the link.
  • stopPropagation() - Stops the Propagation but the Current Target continues.
  • stopImmediatePropagation() - Stops the Propagation and Current Target.

Promise

An object returned by AJAX representing a completion or failure of an event from an asynchronous operation. It handles multiple callback issues that is also used to manage error and success conditions.

Promise States

  • Fulfilled - The operation was successfully completed
  • Rejected - The operation failed
